CPP内存分配情况

发表于 2022-10-21 更新于 2022-12-20 分类于 CPP

C++内存中分为以下几个区：栈区、堆区、全局/静态存储区、常量存储区、代码区。

栈：由编译器管理分配和回收，存放局部变量和函数参数。

堆：由程序员管理，需要⼿动 new/delete malloc/free 进⾏分配和回收，空间较⼤，但可能会出现内存泄漏和空闲碎片的情况。

全局/静态存储区：分为初始化和未初始化两个相邻区域，存储初始化和未初始化的全局变量和静态变量。

常量存储区：存储常量，⼀般不允许修改。

代码区：存放程序的⼆进制代码。

智能指针

发表于 2022-10-18 更新于 2022-12-19 分类于 CPP

为什么需要智能指针

简单来说，智能指针可以帮助我们更好地管理内存空间，避免我们申请动态内存后忘记释放，造成内存泄漏的情况。

需要注意的是，智能指针表面上称为指针，其实是一个类。

智能指针和异常

我们知道，一个异常处理程序能在异常发生后令程序流程继续，这种程序需要确保在异常发生后资源能够被正确释放，简单的方法就是使用智能指针。

当正常处理结束或发生异常，函数退出，局部对象销毁。

weak_ptr

不控制指向对象的生存期，指向一个由shared_ptr管理的对象。weak_ptr不影响shared_ptr的引用计数。如其名。

auto_ptr

auto_ptr具有unique_ptr的部分功能，已经被淘汰，但仍然是标准库的一部分。

auto被淘汰的原因是可能发生潜在的内存泄漏。导致内存泄漏的原因是auto_ptr的赋值运算符重载的方式。

NULL与nullptr的区别

发表于 2022-09-26 更新于 2022-09-28 分类于 CPP

问题引入

之前刷力扣题的时候，就注意到了平常我使用的NULL的地方在CPP题解里大多是已nullptr的形式出现，但因为做题目时，并没有感觉到二者的巨大区别，所以没有细想，但今天又碰到了，于是我决定一探究竟。

强类型语言与弱类型语言

在理解两者区别的时候，我们先来认识一下什么是强类型语言和弱类型语言。

强类型语言中，一旦某一个变量被定义了类型，则如果不经过强制类型转换就无法改变类型，并且一个对象从属的类型必须在编译时刻能够被确定。

弱类型语言则与强类型语言相反，弱类型语言中，一个变量可以根据环境变化自动进行隐式转换就可以改变数据类型。

NULL与nullptr的区别

在C语言中，NULL通常被定义为**#define NULL ((void*)0)，即NULL实际上是一个空指针，在C语言中NULL**被赋给任意一个类型的指针时，发生隐式转换，变成该类型的空指针。

而C++中，NULL的定义是这样的**#define NULL 0*，因为C++不能将void类型的指针隐式转换成其他类型的指针，所以引入了0表示空指针。

这样做会带来一个问题，那就是如果有函数重载的情况那么结果可能会与我们目标不同。

于是为了解决这样的问题，C++11引入了nullptr这个关键字，在任何地方都代表空指针。

总结

省流：在C++中能用nullptr的时候我们还是尽量用nullptr吧！
