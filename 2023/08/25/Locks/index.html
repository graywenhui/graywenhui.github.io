Memory allocator

这个实验是让我们解决锁争用的问题，原代码中我们在多CPU上运行的时候，只有一个空闲列表和一个空闲锁，导致即使是多CPU,实际上也只能有单个CPU能访问到，所以我们要为每个CPU维护一个空闲队列和锁。

修改kinit，这里就是把原来的初始化一个变为为每个CPU都初始化，用到了实验里说到的snprintf来格式化名字。

接着是kfree，按要求里的提示“The function cpuid returns the current core number, but it’s only safe to call it and use its result when interrupts are turned off. You should use push_off() and pop_off() to turn interrupts off and on”，所以我们在获取cpuid的前后要关闭中断和打开中断。这里可能有疑问为什么要在最后再打开中断。7.4节中有提到，如果定时器中断并导致移动到另一个cpu，那这个cpuid将不再正确，所以要求在使用完返回值后再打开中断。

最后是kalloc

这里主要做的修改是else的部分，主要就是在当前cpu上没有空闲内存后就获取别的cpu上的空闲内存。
